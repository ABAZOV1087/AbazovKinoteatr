### 1. Цель работы
Провести автоматизированное тестирование разработанных программных модулей (авторизация, регистрация, капча) с использованием средств Microsoft Visual Studio и фреймворка MSTest.

### 2. Описание реализованных модулей
В ходе работы был проведен рефакторинг кода для обеспечения тестопригодности (Testability):
* **Module Auth**: Публичный метод `Auth(login, password)` для проверки учетных данных в БД.
* **Module Register**: Публичный метод `Register(login, password, name)` для создания новых пользователей.
* **Module Captcha**: Система безопасности, генерирующая случайный код после 3-х неудачных попыток входа.

### 4. Вывод о проведенном тестировании
В ходе выполнения работы все 26 тестов были успешно пройдены. 

**Причины успешного выполнения:**
1. **Рефакторинг архитектуры**: Логика была отделена от интерфейса (Code-behind), что позволило тестировать методы `Auth` и `Register` напрямую.
2. **Изоляция данных**: Применение метода `AsNoTracking()` в Entity Framework позволило тестам работать с актуальным состоянием БД без конфликтов кэширования.
3. **Обработка граничных условий**: Код успешно обрабатывает пустые строки (`IsNullOrWhiteSpace`), слишком длинные значения и попытки дублирования уникальных ключей.
4. **Валидация капчи**: Тесты подтвердили, что доступ к системе блокируется при неверном вводе капчи, даже если логин и пароль верны (после 3 попыток).

### 5. Инструкция по развертыванию
1. Выполните SQL-скрипт `AbazovCinemaDB.sql` на вашем сервере MS SQL Server для создания структуры таблиц и наполнения данными.
2. Откройте решение в Visual Studio.
3. В файле `App.config` (и в основном проекте, и в тестовом) обновите строку подключения `connectionString`, указав ваш `Data Source`.
4. Соберите решение и запустите тесты через «Обозреватель тестов».
